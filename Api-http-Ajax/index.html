Trabalhando com api e requisições HTTP

HTTP é um canal para receber e enviar arquivos ou dados na web

Métodos do HTTP

  • GET

    o Get é o padrão na maioria das vezes, serve para pegar obter as informações de uma url por exemplo como se voce digita-se diretamente no browser

  • POST

    Criar novos usuários e novos registros, exemplo eu pego as informações pelo get altero e envio de volta para cadastrar no back end por exemplo

  • PUT

    Atualizar informações, exemplo tenho usuario de id 70 e troquei esse email, agora preciso enviar pedindo essa troca

  • DELETE

    Serve para deletar informações

  • HEAD

    Serve para puxar o cabeçalho para quando voce precisa dos meta dados (cabeçalho) de algo como o servidor

AJAX

Significa Asynchronous JavaScript e XML, é o uso do objeto XMLHttpRequest para se comunicar com os scripts do lado do servidor atraves de uma API.

Em vez de atualizar a pagina para obter novas informaçẽos do servidor (usando F5 por exemplo) dessa forma forçamos o javascript assincrono para atualizar informações sem recarregar a pagina novamente

PROMISES

São códigos que não vao influenciar na linha do tmepo do JS, basicamente são funções que vão devolver um resultado de sucesso ou erro

Nós quando fazemos uma requisição ela demora um tempo em milisegundos ate a resposta da requisição que pode ter tido resultado positivo ou não

Então quando der errado ou certo minha requisição eu posso fazer alguma lógica com isso

AXIOS

Eh um encapsulamento do XMLHttpRequest que possibilita trabalhar mais facil com as Promises pois sao baseadas em HTTP

Ele eh um biblioteca que pode ser instalada por npm ou via CDN
